Student card variants — single-student surfaces

Four card designs synthesizing the Opus + DeepSeek proposals and the Xiao quotes mining. Each variant shows both the EXPANDED state (landing surfaces like Overview, Schools-first) and the COLLAPSED state (sub-tabs like Tasks, Snapshots, Documents, Meetings, Notes). Pick a direction; field tweaks come after.

Locked (Alex): Module folded INTO the card · Working style chip in HEADER · Citizenship visible · Family/Contacts dropped from card · Two variants (EXPANDED + COLLAPSED) keyed off identity-vs-workspace, not per-surface.
Open: Module placement (folded vs right-column vs strip) · default state (expanded vs compact-with-drawer) · field grouping (4-col blocks vs dense single-row).

B Module as right column — Option b fallback Card stays 3-col · module is matched-height aside
Design idea: Card stays a clean 3-column body (Academic · Hooks · Engagement). Module lives as a right-aligned column matched to card height — always visible, even when the card body collapses on sub-pages. This is the explicit fallback Opus called out: lighter-touch change from today's pattern, easier "no current module" empty state.

  • COLLAPSED: card header only; module aside persists beside the header — module is always at-a-glance regardless of card state
